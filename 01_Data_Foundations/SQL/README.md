# 🗄️ SQL Project: Relational Database Design & Advanced Querying

This folder contains the core SQL projects and assignments for the **Data Foundations** module of the **Master in Big Data, Data Science, & AI**. The focus here is on transforming raw business requirements into robust relational schemas and extracting actionable insights through complex queries.

## 🎯 Project Objectives
The primary goal was to master the lifecycle of data within a RDBMS (Relational Database Management System), focusing on:
1.  **Data Modeling:** Designing Entity-Relationship Diagrams (ERD) and implementing them via DDL (Data Definition Language).
2.  **Data Integrity:** Ensuring consistency through Primary Keys, Foreign Keys, and Constraints.
3.  **Optimization:** Writing efficient DML (Data Manipulation Language) queries to handle large datasets.

---

## 🛠️ Key Skills Applied
In this repository, I demonstrate proficiency in:
* **Complex Joins:** Combining multiple tables (INNER, LEFT, SELF JOINS) to aggregate data.
* **Subqueries & CTEs:** Using Common Table Expressions to simplify logic and improve readability.
* **Window Functions:** Utilizing `RANK()`, `ROW_NUMBER()`, and `OVER(PARTITION BY...)` for advanced ranking and trend analysis.
* **Normalization:** Applying 1NF, 2NF, and 3NF to eliminate redundancy and improve database health.

---

## 📁 Repository Content
* `/scripts`: SQL files containing the creation of schemas and data insertion.
* `/queries`: A collection of business-driven queries (e.g., Sales performance, user behavior, inventory management).
* `/diagrams`: Visual ERD (Entity-Relationship Diagrams) showing the architecture of the databases.

---

## 🚀 Featured Challenge: "The Business Analytics Suite"
One of the main exercises involved creating a database from scratch for a [Events Enterprise].
* **The Problem:** Manage the diversity of events and details, as well as the earnings obtained.
* **The Solution:** I designed a schema to optimize reporting speed and wrote a suite of queries to calculate different data directly in SQL.

### 💡 Why SQL in the age of Big Data?
Even in Big Data and AI environments, SQL remains the universal language for data extraction. Mastering relational logic is essential for feeding clean, structured data into Machine Learning models and BI tools like Tableau.
